## User Story #3

<aside>
👤

Es recomendable trabajar esta story de forma individual. Puede ser asignada a una persona del equipo.

</aside>

***Código:** #main-s1-us3*

***Nombre:** Construir página de registro*

### **Detalle**

Como líder del Equipo de Diseño, quiero que la app disponga de una pantalla de *registro* para permitir que las personas se registren. Esta pantalla será accesible desde /register.

![image.png](attachment:27f2fd03-4160-4e4d-b549-03b1dd8c7ca2:image.png)

### **Escenarios**

- Dado que la persona usuaria debe identificarse para usar la tienda y aún no ha iniciado sesión, cuando llega a la página de registro, y no posee credenciales, debe completar un formulario con su nombre, su email y su contraseña; luego debe poder hacer click en un botón con la leyenda “Registrarse” para iniciar el proceso de registro.

<aside>
✨

***¡BONUS!*** Dado que la persona usuaria ha llegado a la página de registro, y ya está registrada, debe poder clickear en un enlace que la lleve a la página /login.

</aside>

### **Validación**

- La página tiene un input de texto para ingresar el nombre de la persona.
- La página tiene un input de email para ingresar el email de la persona (o nombre de usuario).
- La página tiene un input de tipo contraseña para ingresar la contraseña.
- La página tiene un botón para iniciar el proceso de registro.
- La página tiene un enlace para dirigir la navegación hacia la página de login.
- La página tiene un enlace para dirigir la navegación hacia la página de inicio.
- Se hace uso correcto de las etiquetas HTML y las opciones de accesibilidad.
- Se respetan las buenas prácticas de usabilidad.
