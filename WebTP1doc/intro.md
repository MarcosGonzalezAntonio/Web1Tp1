A medida que aprendemos las bases del desarrollo web con Express, implementaremos un ejercicio cuasi real: una plataforma de ecommerce. Cada paso que demos en Express va a empujar un pedacito más de nuestro *ecommerce*: cada práctico funciona como un sprint, con un objetivo concreto y una entrega clara, igual que en un proyecto real. 

Antes de meternos en el primer sprint, vale presentar el *ecommerce* que vamos a construir: una tienda ficticia, simple pero completa, que nos permita practicar todo lo que una app real necesita. No importa si vende remeras, libros o figuritas; lo importante es que tenga productos, usuarios, carritos, pedidos y un flujo lógico que podamos ir armando paso a paso. 

# Requerimientos del Sprint

El *Equipo de Diseño UI/UX* está encargado de diseñar el nuevo e-commerce de la empresa que busca mejorar la experiencia actual de sus clientes para adquirir productos y servicios de forma más simple, rápida y amigable.

En ese sentido, usando *Design Thinking*, luego de un gran trabajo de *investigación* y análisis de datos y comportamiento, llegaron a un prototipo del nuevo e-commerce que podemos ver en el siguiente link:

Junto con el prototipo, el Equipo de Diseño UI/UX *nos compartió una carpeta con toda la información* para poder aplicar el diseño con código HTML y CSS.

[MiEcommerce - Google Drive](https://drive.google.com/drive/folders/1xs87FmH64hSGP1h6icJUWar2zIfuNOWq?usp=sharing)

Como parte del *Equipo de Desarrollo*, se nos pide implementar estos prototipos para obtener una interfaz funcional para poder realizar las primeras pruebas con un prototipo evolutivo de alta fidelidad.

Con esta idea en mente, dividiremos nuestro trabajo en pequeños pasos dentro de este primer Sprint para poder mostrar la interfaz funcionando en pocos días:

1️⃣  Crear una App con NodeJS y ExpressJS para poder alojar y servir el contenido Web.

2️⃣  Diseñar la estructura HTML de cada pantalla (*vista*)

3️⃣  Aplicar los estilos CSS necesarios para que nuestra app luzca tal como la diseñó el *Equipo de UI/UX*.

4️⃣  Permitir que cada vista sea accesible desde una URL especial.

Para dar cumplimiento a estas tareas y lograr el objetivo del primer sprint sin perder de vista nuestro enfoque en las personas usuarias, vamos a trabajar con ***User Stories*** que podríamos mapear con esta lista de pasos.
